# 人間向けガイド（日本語）

> [!WARNING]
> GoAPI と GoLoader（goloder）はまだ公開していません。

## 目的

このガイドは、このテンプレートを保守・拡張する人向けです。
`.goapi/AGENT.md` の人間版として使ってください。

## このリポジトリの役割

- Minecraft イベントをブリッジ経由で受け取る Go サーバーを提供する
- `modules/` 配下に機能ごとのロジックを分離する
- `main.go` を安定したエントリーポイントとして維持する

## 基本ワークフロー

1. テンプレートを複製する
2. `modules/` に新しい機能ディレクトリを作る
3. `Register()` でイベントとコマンドを登録する
4. `main.go` に登録呼び出しを1行追加する
5. `server` バイナリをビルドして配置する

```bash
go build -o server .
```

## モジュール構成の推奨

```text
modules/
└── myfeature/
    ├── register.go
    ├── command.go
    └── events.go
```

- `register.go`: 登録処理だけを書く
- それ以外のファイル: 実装ロジックを書く

## main.go のルール

- `main.go` はインフラ層として保つ
- 実行基盤の処理を機能モジュールに分散させない
- 追加するのはモジュール登録の1行のみ

## 変更チェックリスト

- 新しいモジュールがコンパイルできる
- `main.go` に登録行が追加されている
- 無関係なインフラ変更を含んでいない
- `server` バイナリが生成できる

## チーム運用メモ

- 機能は小さなモジュール単位で分割する
- ハンドラは決定的でテストしやすく保つ
- 非自明な仕様は実装コードの近くに記録する
