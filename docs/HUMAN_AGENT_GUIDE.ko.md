# 사람용 가이드 (한국어)

> [!WARNING]
> GoAPI와 GoLoader(goloder)는 아직 공개되지 않았습니다.

## 목적

이 문서는 이 템플릿을 유지보수하거나 확장하는 사람을 위한 가이드입니다.
`.goapi/AGENT.md`를 사람 관점으로 정리한 문서입니다.

## 저장소 역할

- 브리지를 통해 전달된 Minecraft 이벤트를 처리하는 Go 서버 제공
- `modules/` 아래 기능별 로직 분리
- `main.go`를 안정적인 진입점으로 유지

## 기본 작업 흐름

1. 템플릿을 복제합니다.
2. `modules/` 아래 새 기능 디렉터리를 만듭니다.
3. `Register()`에서 이벤트/명령을 등록합니다.
4. `main.go`에 등록 호출 한 줄을 추가합니다.
5. `server` 바이너리를 빌드하고 배치합니다.

```bash
go build -o server .
```

## 권장 모듈 구조

```text
modules/
└── myfeature/
    ├── register.go
    ├── command.go
    └── events.go
```

- `register.go`: 등록 코드만 작성
- 기타 파일: 실제 구현 코드

## main.go 규칙

- `main.go`는 인프라 계층으로 유지
- 런타임 초기화 코드를 기능 모듈로 분산하지 않기
- 모듈 등록 한 줄만 추가하기

## 변경 체크리스트

- 새 모듈이 컴파일되는지 확인
- `main.go`에 등록 줄이 있는지 확인
- 무관한 인프라 변경이 없는지 확인
- `server` 바이너리 생성 확인

## 협업 메모

- 작은 단위 모듈을 선호합니다.
- 핸들러는 예측 가능하고 테스트 가능하게 유지합니다.
- 이해가 어려운 동작은 코드 근처에 문서화합니다.
